Engine

Engine represents the connection to the Knowledge Engine’s API.  It exposes that API through two methods, get_object_by_id and get_objects_by_query.

Properties

default-app-id

The app ID to use to find content for in case none is passed into the query.

e.g. com.endlessm.animals-es

default-data-path

The path to the default domains database, if unset will be search for in XDG_DATA_DIRS normally.

e.g.  /endless/share/ekn/data/animals-es

language

The ISO639 language code which will be used for various search features, such as term stemming and spelling correction.

Defaults to empty string

Functions

get_object_by_id

get_object_by_id: function (id,
cancellable,
callback)

Asynchronously fetches an object with ID.

Parameters

idThe unique ID for this object, of the form ekn://domain/sha
cancellableA Gio.Cancellable to cancel the async request.
callbackA function which will be called after the request finished.  The function will be called with the engine and a task object, as parameters.  The task object can be used with get_object_by_id_finish to retrieve the result.

get_object_by_id_finish

get_object_by_id_finish: function (task)

Finishes a call to get_object_by_id.  Returns the ContentObjectModel associated with the id requested, or throws an error if one occurred.

Parameters

taskThe task returned by get_object_by_id

get_objects_by_query

get_objects_by_query: function (query_obj,
cancellable,
callback)

Asynchronously sends a request to xapian-bridge for a given query_obj, and return a list of matching models.

Parameters

query_objA QueryObject describing the query.
cancellableA Gio.Cancellable to cancel the async request.
callbackA function which will be called after the request finished.  The function will be called with the engine and a task object, as parameters.  The task object can be used with get_objects_by_query_finish to retrieve the result.

get_objects_by_query_finish

get_objects_by_query_finish: function (task)

Finishes a call to get_objects_by_query.  Returns both a list of ContentObjectModels and a dictionary with results info.  Throws an error if one occurred.

Members of the info object

more_resultsQueryObject which can be used to get more results for the same query.
upper_boundan upper bound on the total number of results.

Parameters

taskThe task returned by get_objects_by_query
